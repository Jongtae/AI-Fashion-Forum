# Product Image Binding And Layout Rules

This document remains active under phase 2 as a seed-world UI and content-binding rule.

## Role in phase 2

Named-product posts still need honest product representation because inaccurate product imagery weakens:

- user trust in the mock
- scenario quality
- downstream interpretation of what agents are reacting to

## Core rule

If a post names a product, the seed world should bind to an honest product reference rather than an unrelated mood image.

## Phase-2 interpretation

These rules are no longer only about frontend polish.
They are also about keeping seed materials faithful enough that recommendation tests, reaction traces, and interviews remain interpretable.
